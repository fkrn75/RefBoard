## 0. 문서 목적

본 문서는 [PureRef](https://www.pureref.com/)와 **동등한 기능**을 제공하는 데스크탑 레퍼런스 보드 프로그램을 신규 개발하기 위한 기능 명세서다. 추가로, 사용자가 만든 이미지 보드를 **웹으로 공유하여 다른 PC·다른 사람·모바일에서도 열람**하는 기능(PureRef에 없는 차별점)을 설계해 포함한다.

## 1. 제품 개요

### 1.1 PureRef란
화가·3D 아티스트·디자이너가 **레퍼런스(참고) 이미지를 무한 캔버스 위에 자유롭게 배치**해 한눈에 보는 경량 데스크탑 도구. 메뉴 바가 없고, 거의 모든 동작을 **마우스 버튼 + 수식 키(modifier) 조합**과 단축키로 수행하는 독특한 인터랙션이 특징. 작업 중 화면 위에 **항상 띄워두는(Always-on-top)·반투명·클릭 통과** 윈도우 모드가 정체성.

### 1.2 목표
1. PureRef의 핵심 기능을 동등 수준으로 구현한다.
2. **웹 공유**를 1급 기능으로 추가한다 — 보드를 클라우드에 올려 링크 하나로 어디서든 열람.
3. 데스크탑 캔버스 렌더링 엔진과 웹 뷰어를 **같은 코드로 통일**해 유지보수 비용을 낮춘다.

### 1.3 핵심 차별점
| 구분 | PureRef | RefBoard |
|---|---|---|
| 무한 캔버스/자동 배치 | ✅ | ✅ (동등) |
| 항상 위/투명/오버레이 창 | ✅ | ✅ (동등) |
| **웹 공유 링크** | ❌ | ✅ **신규** |
| **모바일 열람** | ❌ | ✅ **신규** |
| **실시간 협업/댓글** | ❌ | 🟢 확장 |
| 가격 | PWYW/상업 유료 | (미정) |

### 1.4 대상 사용자
- 1차: 일러스트레이터, 컨셉 아티스트, 3D 모델러 (PureRef 기존 사용자층)
- 2차(웹 공유 수혜): 클라이언트에게 무드보드를 보내는 디자이너, 팀 협업, 외부 검수자, 모바일에서 보고 싶은 사용자

## 2. 기능 명세 — PureRef 동등 기능

> 단축키는 PureRef 관례를 기본값으로 채택하되(전부 재바인딩 가능 설계), 신규 제품이므로 자체 조정 가능. `LMB`=좌클릭, `MMB`=중간 버튼.

### 2.1 이미지 가져오기 (Import)

| 기능 | 설명 | 우선 | 기본 단축키 |
|---|---|---|---|
| 드래그 앤 드롭 | 탐색기/브라우저에서 이미지·다중 파일·폴더를 캔버스로 끌어다 놓기 | 🔴 P0 | (드래그) |
| 클립보드 붙여넣기 | 복사된 이미지/스크린샷 즉시 삽입 | 🔴 P0 | `Ctrl+V` |
| 파일 열기 | 파일 브라우저로 이미지 선택 가져오기 | 🔴 P0 | `Ctrl+I` |
| URL 붙여넣기 → 페이지 스크랩 | URL을 붙이면 **그 웹페이지의 모든 이미지를 수집** | 🟡 P1 | `Ctrl+V` |
| 폴더 일괄 가져오기 | 폴더 통째로 대량 삽입 | 🟡 P1 | (드래그) |
| 내장 스크린샷 캡처 | OS 캡처 의존이 아닌 **자체 영역 캡처** (PureRef엔 없음 → 차별점) | 🟢 P2 | (지정) |
| 이미지 교체(Replace) | 기존 이미지를 위치·변형 유지한 채 다른 파일로 교체 | 🟢 P2 | (지정) |
| 누락 이미지 재연결(Relink) | 링크 모드에서 끊긴 원본 경로 재지정 | 🟡 P1 | `Ctrl+Alt+Shift+R` |

### 2.3 이미지 변형 — 비파괴(Transform)

> **모든 변형은 원본 픽셀을 건드리지 않는다.** 데이터 구조 = `원본 이미지` + `변형행렬(위치/스케일/회전)` + `크롭 사각형` 별도 보존. Reset crop ≠ Reset transform (각각 독립).

| 기능 | 설명 | 우선 | 기본 단축키 |
|---|---|---|---|
| 이동 | 드래그 이동 | 🔴 P0 | `LMB` |
| 축 고정 이동 | 수평/수직만 | 🟡 P1 | `Shift+LMB` |
| 이웃 스냅 이동 | 인접 이미지 가장자리에 자석 스냅 | 🟢 P2 | `Shift+Space+LMB` |
| 크기 조절(Scale) | 기즈모/단축키, 모서리 드래그 | 🔴 P0 | `Ctrl+Alt+LMB` |
| 중심 기준 스케일 | 중심점 고정 스케일 | 🟡 P1 | 기즈모+`Alt` |
| 회전 | 자유 회전 | 🟡 P1 | `Ctrl+LMB` |
| 45° 스냅 회전 | 각도 스냅 | 🟢 P2 | `Ctrl+Shift+LMB` |
| 좌우/상하 뒤집기 | Flip H / V | 🟡 P1 | `Alt+Shift+H` / `V` |
| 자르기(Crop) | 비파괴 크롭, 영역 지정·내부 팬/줌 | 🟡 P1 | `Ctrl+Alt+Shift+C` |
| 크롭 초기화 | 원본 형태 복원 | 🟡 P1 | `Ctrl+Shift+C` |
| 변형 초기화 | 크기·회전만 복원 | 🟡 P1 | `Ctrl+Shift+T` |
| 복제 | 선택 복제 | 🔴 P0 | `Ctrl+D` |
| 삭제 | 캔버스에서 제거 | 🔴 P0 | `Del` |
| 픽셀 보간 토글 | bilinear ↔ nearest(픽셀아트용) | 🟢 P2 | `Alt+T` |

### 2.4 배치 & 정렬 (Arrange · Align · Distribute)

> **자동 패킹(bin-packing)이 PureRef의 킬러 기능.** 창 크기에 맞춰 모든 이미지를 빈틈없이 자동 배치.

| 기능 | 설명 | 우선 | 기본 단축키 |
|---|---|---|---|
| 최적 자동 배치(Pack) | 창 크기에 맞춰 자동 패킹 | 🔴 P0 | `Ctrl+P` |
| 캔버스 최적화 | 모든 이미지에 캔버스를 딱 맞게 조임 | 🟡 P1 | `Ctrl+O` |
| 정렬(이름/추가순/경로/순서/랜덤) | 기준별 격자 정렬, 재실행 시 역순 토글 | 🟡 P1 | `Ctrl+Alt+N/A/D/O/R` |
| 가장자리 정렬(Align) | 상/하/좌/우 밀착 | 🟡 P1 | `Ctrl+←/→/↑/↓` |
| 균등 분배(Distribute) | 수평/수직 균등 간격 | 🟡 P1 | `Ctrl+Alt+Shift+↑/↓` |
| 크기 정규화 | 동일 크기/너비/높이/스케일/면적 | 🟢 P2 | `Ctrl+Alt+화살표` |
| 쌓기(Stack) | 한 자리에 포개기 | 🟢 P2 | `Ctrl+Alt+S` |
| 정렬 간격(Padding) | 정렬 시 이미지 간 픽셀 간격 설정 | 🟡 P1 | (설정) |

## 3. 추가 기능 — 웹 공유 (핵심 차별점)

> 사용자 요청: *"만들어진 이미지 보드를 웹상에 공유해서 다른 컴이나 다른 사람, 모바일에서도 볼 수 있는 기능."*
> PureRef에 **없는** 기능으로, 본 제품의 핵심 차별점이자 시장 경쟁력.

### 3.1 사용자 시나리오

1. **클라이언트 공유**: 디자이너가 무드보드를 만들고 "웹에 공유" → 생성된 링크를 카톡/메일로 전달 → 클라이언트가 **설치 없이** 브라우저로 열람.
2. **멀티 디바이스**: 작업 PC에서 만든 보드를 노트북·태블릿·**스마트폰**에서 그대로 확인.
3. **팀 협업(확장)**: 여러 명이 같은 보드에 댓글/핀을 달고, 데스크탑 수정이 웹 뷰어에 실시간 반영.

### 3.2 기능 요구사항

| 기능 | 설명 | 우선 |
|---|---|---|
| 보드 → 클라우드 업로드 | 데스크탑에서 "웹에 공유" 클릭 → 이미지+레이아웃 업로드 | 🔴 P0 |
| 공유 링크 생성 | 고유 단축 URL 발급 (예: `refboard.app/b/xR3k9a`) | 🔴 P0 |
| 웹 뷰어(읽기 전용) | 브라우저에서 보드 줌/팬 열람 | 🔴 P0 |
| 모바일 반응형 | 터치 제스처(핀치 줌/드래그 팬), 작은 화면 최적화 | 🔴 P0 |
| 이미지 라이트박스 | 개별 이미지 탭 → 풀스크린 + 다음/이전 | 🟡 P1 |
| 이미지 다중 해상도 | 썸네일/중간/원본 점진 로딩(LQIP) | 🟡 P1 |
| 공유 갱신(재업로드) | 보드 수정 후 같은 링크로 업데이트 | 🟡 P1 |
| 권한 설정 | 공개/비공개 토큰/비밀번호/만료 | 🟡 P1 |
| 다운로드 허용·금지 | 원본 저장 가능 여부 | 🟡 P1 |
| PWA 설치 | 모바일 홈화면 추가, 오프라인 캐싱 | 🟢 P2 |
| 실시간 동기화 | 데스크탑 수정 → 웹 자동 반영(WebSocket) | 🟢 P2 |
| 댓글·핀 협업 | 이미지에 댓글/위치 핀 | 🟢 P2 |
| 동시 커서 | Figma식 멀티 커서 | ⚪ P3 |
| 임베드 코드 | `<iframe>`으로 외부 사이트 삽입 | 🟢 P2 |

### 3.3 시스템 아키텍처

```
┌─────────────────┐   업로드(이미지+보드 JSON)   ┌──────────────────────┐
│  RefBoard 데스크탑 │ ───────────────────────────▶ │   백엔드 API          │
│  (보드 편집)      │ ◀─── 공유 링크/상태 ──────────│  (인증·업로드·발급)    │
└─────────────────┘                               └─────────┬────────────┘
                                                            │ 저장
                                              ┌─────────────┴─────────────┐
                                              │ 오브젝트 스토리지(이미지)    │
                                              │ + DB(보드 메타/권한)        │
                                              └─────────────┬─────────────┘
                                                            │ fetch(보드 JSON + 이미지)
┌──────────────────────────┐   공유 링크 접속   ┌──────────┴─────────────┐
│ 브라우저 (PC/모바일/태블릿) │ ───────────────▶ │  웹 뷰어 SPA (읽기 전용) │
│  - 줌/팬, 라이트박스        │                  │  = 데스크탑과 동일 렌더 엔진 │
└──────────────────────────┘                  └────────────────────────┘
```

**핵심 설계 원칙**: 데스크탑 캔버스 렌더러와 웹 뷰어 렌더러를 **동일한 코드(공유 패키지)** 로 만든다. 보드 상태(JSON)는 양쪽 공통 스키마. → "한 번 만들면 데스크탑·웹 동시 지원".

### 3.4 웹 뷰어 기능

- 줌/팬: 마우스 휠·드래그 + 모바일 핀치/스와이프
- 전체 보기(fit) / 개별 이미지 포커스
- 이미지 라이트박스(풀스크린, 다음/이전, 핀치 줌)
- 미니맵(선택)
- 보드 제목·설명·작성자 표시
- 읽기 전용(편집 불가) — 단, 댓글은 권한에 따라 허용
- 그레이스케일 토글 등 가벼운 보기 옵션(선택)

### 3.5 권한 & 보안

| 모드 | 설명 |
|---|---|
| 공개 링크 | 링크만 있으면 누구나 |
| 비공개(추측 불가 토큰) | 충분히 긴 랜덤 URL |
| 비밀번호 보호 | 열람 시 암호 입력 |
| 만료 기간 | N일 후 자동 비활성 |
| 다운로드 금지 | 우클릭/원본 접근 제한(완전 차단은 불가, 억제 수준) |
| 워터마크(선택) | 공유 이미지에 워터마크 |

- 전송 구간 HTTPS, 저장 시 비공개 버킷 + 서명 URL(signed URL).
- 저작권 고려: 사용자가 올린 이미지의 권리는 사용자 책임 — 약관 명시.

### 3.6 모바일 / PWA

- 반응형 레이아웃(세로/가로), 터치 우선 인터랙션.
- PWA: 홈 화면 추가, 서비스 워커로 본 보드 오프라인 캐싱.
- 저사양 기기 대비: 해상도별 이미지 + lazy-load + 가상화(뷰포트 밖 언로드).

## 4. 데이터 모델

### 4.1 보드 스키마 (JSON, 데스크탑·웹 공통)

```jsonc
{
  "schema": "refboard/1.0",
  "board": {
    "id": "xR3k9a",
    "title": "캐릭터 컨셉 무드보드",
    "canvas": { "bg": "#1e1e1e", "grid": { "type": "dots", "snap": true } },
    "camera": { "x": 0, "y": 0, "zoom": 1.0 }
  },
  "items": [
    {
      "id": "img_01",
      "type": "image",
      "src": "assets/img_01.webp",      // 임베드 시 base64 또는 내부참조
      "natural": { "w": 1920, "h": 1080 },
      "transform": { "x": 120, "y": 80, "scale": 0.5, "rotation": 0 },
      "crop": { "x": 0, "y": 0, "w": 1920, "h": 1080 },
      "opacity": 1.0, "grayscale": false, "locked": false, "z": 3,
      "comment": "참고: 의상 실루엣"
    },
    { "id": "note_01", "type": "note", "html": "<b>색감</b> 한류 톤", "transform": {} },
    { "id": "grp_01", "type": "group", "children": ["img_01", "note_01"] }
  ]
}
```

### 4.2 파일 포맷 `.refb`

- 컨테이너: ZIP 기반(`board.json` + `assets/` 이미지들 + 썸네일). 검증·디버깅 용이.
- 임베드 모드: 이미지 원본을 `assets/`에 포함. 링크 모드: `src`에 외부 절대경로.
- 버전 필드로 상위 호환 관리.

### 4.3 클라우드 보드 스키마(서버)

- `boards` 테이블: id, owner, title, visibility, password_hash, expires_at, created/updated.
- 이미지: 오브젝트 스토리지에 `boards/{id}/{itemId}_{size}.webp` (size = thumb/medium/orig).
- 보드 JSON에서 `src`를 스토리지 키로 치환해 서빙.

## 5. 기술 스택 권장안

> 웹 공유 요구사항 때문에 **렌더링을 웹 기술로 통일**하는 것이 결정적으로 유리하다. 데스크탑 캔버스와 웹 뷰어가 같은 엔진을 공유한다.

### 5.1 데스크탑 앱
| 후보 | 장점 | 단점 | 추천도 |
|---|---|---|---|
| **Tauri 2.0** (Rust + 웹FE) | 초경량(수 MB), 빠름, 웹 코드 재사용, 크로스플랫폼 | Rust 학습 곡선 | ★★★ 권장 |
| Electron | 가장 쉬움, 생태계 풍부 | 무겁고 메모리 큼 | ★★ |
| 네이티브(C#/Qt) | 최고 성능 | 웹 뷰어 별도 개발 → 코드 중복 | ★ |

### 5.2 렌더링 엔진
- **PixiJS(WebGL)** 권장 — 수백 장 이미지를 60fps로. 줌/팬/변형 행렬에 적합.
- 대안: HTML5 Canvas 2D(단순), 또는 Konva.js(객체 모델 편리).

### 5.3 백엔드 & 스토리지
- 서버리스 권장: **Cloudflare Workers + R2(이미지) + D1/KV(메타)** — 저비용·글로벌 CDN.
- 대안: Node(Hono/Express) + S3 + Postgres.
- 이미지 처리: 업로드 시 sharp/wasm으로 WebP·다중 해상도 생성.

### 5.4 코드 재사용 전략 (핵심)
```
packages/
  core-canvas/   ← 보드 상태·변형·렌더(PixiJS)  [데스크탑 + 웹 공유]
  desktop/       ← Tauri 셸, 파일 I/O, 윈도우 모드
  web-viewer/    ← 읽기 전용 뷰어(core-canvas 재사용)
  server/        ← 업로드 API, 권한, 이미지 파이프라인
```
모노레포로 `core-canvas`를 데스크탑·웹이 공유 → 기능 추가 1회로 양쪽 반영.

## 9. 부록 B — PureRef 대비 차이·주의점

1. **`pureref.com/manual.php`는 폐기(404)** → 현행 문서는 `pureref.com/handbook/...`.
2. **z-order 2종**(캔버스 레이어 vs OS 창 스태킹)을 코드에서 분리.
3. **모든 변형은 비파괴** — 원본+변형행렬+크롭 분리 저장.
4. **자동 패킹**(bin-packing)이 정체성 — 알고리즘 품질이 체감 품질 좌우.
5. **URL 붙여넣기 = 페이지 이미지 스크랩** (CORS·저작권 주의).
6. **마우스 통과/항상 위** 등 OS 윈도우 제어는 플랫폼별 구현 필요(Tauri 플러그인/네이티브 API).
7. **가격/라이선스 변동** — PureRef는 PWYW(개인)+상업 유료. RefBoard 가격 정책 별도 결정.
8. 웹 공유의 **저작권·약관** 명문화 필요(사용자 업로드 책임).
